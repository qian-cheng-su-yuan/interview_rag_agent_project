# 面试讲解要点

## 1. 一句话介绍

这是一个面向企业内部资料问答场景的 AI Agent / RAG 项目。系统可以上传文档，自动切分并建立索引，用户提问后先检索相关资料，再调用大模型生成回答，同时展示参考片段。

## 2. 为什么要做 RAG

直接让大模型回答企业内部问题，会有两个问题：

1. 大模型不知道企业内部资料；
2. 模型可能编造答案。

RAG 的作用是先从企业知识库中检索相关资料，再让模型基于资料回答，从而增强答案的依据性。

## 3. 核心链路怎么实现

项目核心代码在 `app/services/rag_service.py`：

```text
问题输入 → 向量检索 → 拼接上下文 → 调用大模型 → 返回答案和参考片段
```

## 4. 项目为什么这样分层

- `api`：只处理接口入参和返回。
- `schemas`：定义请求和响应数据结构。
- `services`：写核心业务逻辑。
- `core`：放配置和日志。
- `utils`：放通用工具函数。

这种写法比把所有代码写在一个文件里更容易维护，也更接近企业项目。

## 5. 没有用真正向量数据库怎么办

当前版本为了方便面试演示，用 scikit-learn 的 TF-IDF 做轻量检索。这样可以降低运行门槛。后续可以把 `LocalVectorStore` 替换成 Chroma、FAISS 或 Milvus，业务层不用大改。

## 6. Prompt 怎么控制幻觉

Prompt 中明确要求模型：

- 优先依据知识库片段回答；
- 没有依据时说明无法确定；
- 不允许编造企业制度。

同时前端展示参考片段，便于用户验证答案来源。

## 7. 项目可扩展点

- 多轮对话 Memory；
- Tool Calling；
- 多知识库隔离；
- 用户登录和权限控制；
- 文档增量索引；
- Docker 部署；
- 接入企业微信或飞书机器人。

## 8. 面试官可能追问

### Q：为什么不用 LangChain？

A：这个项目把 RAG 关键流程手写出来，是为了更清楚地展示自己理解每一步。后续如果要接入更多工具和链式调用，可以再引入 LangChain 或 LlamaIndex。

### Q：为什么不用 Chroma / FAISS？

A：本地演示版本优先保证可运行和低依赖。向量库已经通过 `LocalVectorStore` 做了边界封装，后续替换成 Chroma / FAISS 主要改这一层。

### Q：如何降低模型幻觉？

A：一是检索阶段只给模型提供相关片段；二是 Prompt 要求没有依据时说明无法确定；三是返回参考片段，方便用户追溯；四是后续可以增加答案置信度和人工反馈。

### Q：项目如何部署到服务器？

A：后端可以用 Uvicorn/Gunicorn 部署，前端 Streamlit 可以单独运行，也可以改成 Vue/React。正式部署时会加 Docker、Nginx、日志采集和权限认证。
